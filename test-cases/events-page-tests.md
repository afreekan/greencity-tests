# Test Cases: Сторінка Events

## Test Case 1: Перегляд деталей існуючої події
**Title:** Перевірка відображення детальної інформації про обрану подію  
**Preconditions:** 1. Користувач не авторизований (або авторизований).
2. Користувач знаходиться на сторінці [Events](https://www.greencity.cx.ua/#/greenCity/events).
3. На сторінці відображається щонайменше одна картка події.

| Step | Action | Data | Expected Result |
| :--- | :--- | :--- | :--- |
| 1 | Знайти картку будь-якої події у загальному списку та клікнути на неї. | N/A | Сторінка завантажується, відбувається перехід на сторінку деталей обраної події. |
| 2 | Перевірити наявність основних елементів на сторінці: Заголовок, Дати/Час, Локація, Опис, Зображення. | N/A | Вся інформація відображається коректно і повністю відповідає коротким даним, що були на картці-прев'ю. |
| 3 | Натиснути кнопку "Back to events". | N/A | Відбувається повернення на загальну сторінку зі списком подій. |

## Test Case 2: Робота фільтра сайта за локацією користувача

| Step | Action | Data | Expected Result |
| :--- | :--- | :--- | :--- |
| 1 | Натиснути на панель фільтрів у блоці "Location". | N/A | Блок фільтрів присутній і доступний для взаємодії. |
| 2 | Написати у фільтрі локацію "Київ". | Місто: `Kyiv` | Список подій автоматично оновлюється. |
| 3 | Перевірити локацію у знайдених картках подій. | N/A | Відображаються лише ті події, у яких локація вказана як "Kyiv". |
